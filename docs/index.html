The hardware

The 2026 ExpertBook Ultra is a Panther Lake machine with three Linux-side issues out of the box: a brand-new haptic touchpad whose pressure axis confuses libinput, Cirrus speaker amplifiers whose firmware isn't shipped for this exact subsystem ID, and a Wi-Fi 7 BE211 driven by the new iwlmld driver.

What gets fixed

Three independent modules, each delivering a targeted patch. Mix and match.

touchpad-fix v1.1.0

The kernel mis-reports the haptic pressure-axis range, so libinput discards every touch as a "kernel bug: Touch jump." Cursor never moves.

udev hwdb libinput quirk

  • Clamps ABS_MT_PRESSURE to a sane range
  • Tells libinput to ignore pressure entirely (AttrEventCode)
  • Same pattern as the shipped Asus UX302LA quirk

audio-fix v1.2.0

CS35L56 amps boot in FIRMWARE_MISSING; ALSA UCM is missing the cs42l43-spk+cs35l56 codec dir; WirePlumber lands on an unrouted stereo-fallback profile.

CS35L56 firmware UCM init.conf WirePlumber rule

  • Bin + wmfw for subsystem 1043:15e4
  • Patches chip ROM 3.4.4 → 3.13.4, calibration applies
  • Pins the card to Pro Audio so the Speaker sink is reachable
  • Hides Bluetooth/Deepbuffer noise; renames sinks/sources

wifi-fix v1.1.0

"Missed beacons exceeds threshold" spam, occasional Microcode SW error freezes, drops on 6 GHz / 320 MHz. Pure userspace tunables — no band downgrade, Bluetooth coex stays on.

modprobe.d tmpfiles.d NM dispatcher

  • iwlmld.power_scheme=1 (active, no driver power-save)
  • Global PCIe ASPM = performance (CNVi has no per-device knobs)
  • Disables TSO/GSO/GRO on iwlwifi (offload bug workaround)

Install

Three modules, one patcher. Idempotent install, tracked versions, previewable diffs, clean uninstall.

  1. Clone the repo: 
    git clone https://github.com/burakgon/asus-expertbook-linux.git
cd asus-expertbook-linux
  2. Inspect what's available: 
    ./patch.sh list
  3. Install everything: 
    ./patch.sh install-all
sudo reboot
  4. Or run the interactive menu (re-execs under sudo automatically): 
    ./patch.sh

After reboot, verify: 

./patch.sh status

Modules

Each module is a folder containing a module.sh manifest, the payload files, and a README. The patcher discovers modules automatically — drop a new folder in to add one.

ModuleVersionDescription
touchpad-fix 1.1.0 PixArt 093A:4F05 haptic touchpad — pressure quirk + libinput override
audio-fix 1.2.0 Cirrus CS35L56 firmware + UCM + WirePlumber Pro Audio profile
wifi-fix 1.1.0 Intel Wi-Fi 7 BE211 — iwlmld power, ASPM, offload tweaks

The patcher

patch.sh is a small bash module manager. Each module declares MODULE_FILES, MODULE_VERSION, and optional pre/post hooks. The patcher records the installed version under /var/lib/asus_expertboot_patcher and flags updates whenever the source MODULE_VERSION bumps.

Commands

./patch.sh                # interactive menu
./patch.sh list           # discover + state
./patch.sh status         # detailed
./patch.sh diff           # source vs installed
./patch.sh install <mod>  # idempotent (also "update")
./patch.sh update-all     # only re-installs out-of-date modules
./patch.sh uninstall <mod>

FAQ

Why not just upstream the fixes?

They will be — these are workarounds for upstream gaps. The touchpad needs a libinput quirk and a kernel HID parser fix. The audio needs cs35l56-b0-dsp1-misc-104315e4-l2u{0,1}.{wmfw,bin} shipped in linux-firmware and a real cs42l43-spk+cs35l56 codec dir in alsa-ucm-conf. The Wi-Fi tweaks should become driver defaults once the iwlmld stack matures. When all three land upstream, this whole repo becomes deletable.

Does this break Bluetooth?

No. The Wi-Fi module deliberately leaves iwlwifi.bt_coex_active=Y alone. BT audio, HID, and file transfer keep working as usual.

Does this downgrade Wi-Fi 7 to Wi-Fi 6?

No. The card stays on Wi-Fi 7 / 6 GHz / 320 MHz wherever the AP and signal allow it. The wifi-fix only disables the driver-side power-save loop, the L1.x ASPM wake latency, and the buggy TX-segmentation offload — none of which limits link speed.

What about the F1 mute LED?

Still stays in the EC firmware default. The UCM SetLED hook only fires under a HiFi UCM profile, and we route through Pro Audio because the HiFi profile for cs42l43-spk+cs35l56 doesn't exist upstream yet. Cosmetic, doesn't affect actual audio.

Will this work on similar ASUS models?

The audio-fix's firmware files are matched on PCI subsystem 1043:15e4 (this exact laptop). Sibling models 104315d4 and 104315f4 would need their own firmware bins — but the rest of the patcher (touchpad and Wi-Fi) is largely transferable. PRs welcome.

Where does the CS35L56 firmware come from?

Both .bin files are upstream linux-firmware (cirrus/cs35l56-b0-dsp1-misc-104315e4-l2u{0,1}.bin). The .wmfw is the generic cs35l56/CS35L56_Rev3.13.4.wmfw renamed into the per-OEM filename pattern the cs35l56 driver looks for. The dmesg banner shows the firmware was built by Cirrus for ASUS B9406CAA.

Add my model / adjacent fix?

Open an issue or a PR. The module format is intentionally tiny — a folder with module.sh, the files, and a README. Look at touchpad-fix/module.sh as the smallest example.
